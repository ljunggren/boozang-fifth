# Documentation

## READMEs

- **Root README**: Always maintain a main README file in the project root for GitHub. It should include the project overview, directory structure, setup instructions, and development workflow.
- **Directory READMEs**: Every significant directory should have a `README.md` explaining its purpose, contents, and conventions. Generate READMEs as you build out new directories.

When creating a new directory as part of a task, always include a README.md for it.

## Other Documentation

- **Diagrams**: Use Mermaid for sequence diagrams, architectural overviews, and data flow diagrams.
- **Inline docs**: Use JSDoc/TSDoc for complex functions and public APIs. Don't over-document obvious code.
